# kg10key (Custom Macro Pad)

Bambu Lab A1 miniで出力したケースと、カスタム基板（PCB）を組み合わせた自作テンキー/マクロパッド・プロジェクトです。

## プロジェクト開始日
* 2026年3月7日

## 概要

`kg10key`は、実用性と拡張性を兼ね備えた10キー配列のマクロパッドです。
* ベースレイヤーでの数字入力に加え、レイヤー切り替えによって
* 動画編集やコーディングのショートカットを効率化することを目的に設計されています。

## 特徴・スペック

* **ケース:** Bambu Lab A1 miniにて3Dプリント（自作設計）
* **制御:** カスタムPCB（設計中・発注予定）
* **インターフェース:** ロータリーエンコーダー搭載（音量調節やタイムライン操作に対応）
* **ファームウェア:** QMK / VIA 対応（予定）
* **レイヤー構成:**
    * **Layer 0:** 標準テンキー入力
    * **Layer 1:** 動画編集ショートカット / カスタムマクロ

## リポジトリ修正の覚書 (2026-03-23)

リポジトリ修正の覚書 (2026-03-23)
分割キーボード設定から、単体テンキー（kg10key）としての最小構成へ全面移行しました。

1. シールド定義の単一化 (Kconfig.shield)
左右分離設定の廃止: _LEFT / _RIGHT の分割定義をすべて削除。

単一定義への集約: SHIELD_KG10KEY として定義し、ビルド時に迷いなく認識される構成に変更。

2. デバイスツリーの整理 (.overlay, .dtsi)
レイヤー役割の分離: chosen ノード（スキャン選択）を .overlay に、ピンアサインやマトリックス構造を .dtsi に分けることで保守性を向上。

物理的整合性の確保: マトリックス（3列×4行）に対し、不要なノード（左右結合用のオフセット等）を排除した最短の定義を適用。

3. コンボによる確実なレイヤー切り替え (.keymap)
Hold-Tapの廃止: 以前の長押しによるレイヤー移行が不安定だったため、より確実な**コンボ（同時押し）**による遷移へ完全移行。

誤爆防止のタイムラグ設定:

timeout-ms = <80>;: 同時押しの猶予を80msに設定。

require-prior-idle-ms = <150>;: 直前の打鍵から150ms空いた場合のみコンボを有効にし、高速タイピング中の意図しないレイヤー移動を防止。

直感的な操作配置:

L0 (7+8), L1 (8+9), L2 (7+4) の配置で各モードに瞬時にアクセス。

7+8 (2+3) の同時押しに Backspace を割り当て、指の移動距離を短縮。
